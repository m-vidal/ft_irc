/msg ou /privmsg com varios destinatarios

{
	não pode começar com número

	não pode conter espaços

	não pode conter caracteres proibidos

	tamanho razoável

	432 ERR_ERRONEUSNICKNAME
}

Boa ideia testar com **irssi + nc ao mesmo tempo** — isso encontra muitos bugs em ft_irc. Vou organizar os testes por comando e indicar **como executar**.

Assumindo:

```text
server port: 6667
password: pass123!
```

---

# 1. PASS

### PASS correto

nc:

```
PASS pass123!
```

esperado: **nenhuma resposta** (IRC padrão não responde PASS)

---

### PASS errado

```
PASS wrongpass
```

esperado:

```
464 * PASS :Wrong password!
```

---

### PASS duas vezes

```
PASS pass123!
PASS pass123!
```

esperado:

```
462 * PASS :User already registered!
```

---

# 2. NICK

### nick válido

```
NICK marc
```

esperado: sem resposta (até USER completar registro)

---

### nick começando com número

```
NICK 123abc
```

esperado:

```
432 * NICK :Erroneous nickname
```

---

### nick com caractere proibido

```
NICK mar:c
```

esperado:

```
432 * NICK :Erroneous nickname
```

---

### nick já em uso (2 clientes)

cliente 1:

```
NICK marc
```

cliente 2:

```
NICK marc
```

esperado no cliente 2:

```
433 marc :Nickname is already in use
```

---

# 3. USER

### USER correto

```
USER marc localhost server :Marc Silva
```

esperado após PASS + NICK:

```
001 marc :Welcome to ircserv
002 marc :Your host is ircserv
003 marc :This server was created
004 marc :ircserv version
```

---

### USER com poucos argumentos

```
USER marc localhost
```

esperado:

```
461 USER :Not enough parameters!
```

---

# 4. PING

nc:

```
PING 12345
```

esperado:

```
:ircserv PONG ircserv :12345
```

---

irssi automaticamente manda PING.

verifique se responde com:

```
PONG
```

senão irssi desconecta.

---

# 5. JOIN

### criar canal

```
JOIN #test
```

esperado:

```
:nick!user@host JOIN #test
353 nick = #test :nick
366 nick #test :End of /NAMES list
```

---

### entrar em canal existente

cliente 1:

```
JOIN #test
```

cliente 2:

```
JOIN #test
```

cliente 1 deve receber:

```
:nick2!user@host JOIN #test
```

---

### JOIN sem

```
JOIN test
```

esperado:

```
403 JOIN :No such channel
```
